<Callout type="info">
One-click database export: DDL + data as batched INSERT statements. Configurable row limits for dev/test datasets.
</Callout>

## What Gets Exported

- **DDL** — `CREATE TABLE` statements with columns, types, defaults, constraints, and indexes
- **Data** — Row data as batched `INSERT` statements

## Batched INSERT Statements

Multiple rows are grouped into a single statement for smaller file size and faster imports:

```sql
INSERT INTO users (id, name, email) VALUES
  (1, 'Alice', 'alice@example.com'),
  (2, 'Bob', 'bob@example.com'),
  (3, 'Carol', 'carol@example.com');
```

## Configurable Row Limit

Set a maximum number of rows to export per table. Useful for:

- Exporting structure with sample data only
- Working with very large tables where full export is impractical
- Creating lightweight dev/test datasets

Set to `0` or leave blank to export all rows.

## Usage

<Steps>
  <Step>
    ### Select Database
    Choose the connection and database to export.
  </Step>
  <Step>
    ### Configure Options
    Set the row limit (optional).
  </Step>
  <Step>
    ### Choose Output Location
    Select where to save the export file.
  </Step>
  <Step>
    ### Start Export
    The generated `.sql` file can be re-imported using the [SQL File Execution](/en/docs/sql-file) feature.
  </Step>
</Steps>
