<Callout type="info">
Execute `.sql` files directly with smart statement splitting (handles functions, triggers, quoted strings), real-time progress tracking, and cancellation.
</Callout>

## Running a SQL File

<Steps>
  <Step>
    ### Open the Panel
    Open the SQL File Execution panel.
  </Step>
  <Step>
    ### Select File
    Choose a `.sql` file from your filesystem.
  </Step>
  <Step>
    ### Choose Target
    Select the target connection and database.
  </Step>
  <Step>
    ### Execute
    Click execute and monitor progress in real time.
  </Step>
</Steps>

## Smart Statement Splitting

DBX parses SQL files intelligently rather than naively splitting on semicolons:

- **Functions and procedures** — `BEGIN...END` blocks are treated as a single unit
- **Triggers** — `CREATE TRIGGER` statements with embedded logic are kept intact
- **Quoted strings** — Semicolons inside string literals are not treated as delimiters

## Cancelling Execution

Click the cancel button to stop execution before the file finishes. Statements already executed are not rolled back — only subsequent statements are skipped.
