<Callout type="info">
Edit table structure visually — no `ALTER TABLE` by hand. All changes are previewed as SQL before execution.
</Callout>

## Column Editor

Manage columns through a visual interface:

- **Add column** — Set name, type, nullable, default, and comment
- **Modify column** — Rename, change type, adjust defaults
- **Delete column** — Mark for removal

All changes are staged locally until you confirm.

## Index Management

Manage indexes in the same interface:

- Create new indexes (unique, non-unique) with selected columns
- Modify existing index configurations
- Remove indexes

## SQL Preview

<Callout type="info">
Before applying, DBX shows the exact `ALTER TABLE` statements — `ADD COLUMN`, `MODIFY COLUMN`, `DROP COLUMN`, `CREATE INDEX`, etc. Review and confirm before committing.
</Callout>

## Database Support

| Engine | Support |
|---|---|
| MySQL | Full |
| PostgreSQL | Full |
| SQLite | Full |
| SQL Server | Full |

Behavior adapts to the capabilities of each engine. For example, SQLite's limited `ALTER TABLE` support is handled transparently by DBX.
