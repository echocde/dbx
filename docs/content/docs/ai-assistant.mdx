<Callout type="info">
Built-in AI assistant — no plugins needed. Describe what you want in plain language, and AI generates SQL, optimizes queries, and fixes errors.
</Callout>

DBX's AI assistant is deeply integrated with your database metadata. It knows your tables, columns, and relationships, so the SQL it generates actually works with your schema.

![DBX AI Assistant](/screenshot-ai.jpg)

## Setup

<Steps>
  <Step>
    ### Open Settings
    Click the **Settings** icon in the toolbar.
  </Step>
  <Step>
    ### Choose AI Provider
    Go to **AI Settings** and select your provider.
  </Step>
  <Step>
    ### Enter API Key
    Enter your API key (and endpoint URL for custom providers).
  </Step>
  <Step>
    ### Select a Model
    Pick a model and you're ready to go.
  </Step>
</Steps>

## Core Features

### Natural Language to SQL

Describe what you want to query in plain language. The AI generates SQL based on your actual database schema:

> **You say:** "Show me the top 10 customers by total order amount in the last month"
>
> **AI generates:**
> ```sql
> SELECT c.name, SUM(o.amount) AS total_amount
> FROM customers c
> JOIN orders o ON c.id = o.customer_id
> WHERE o.created_at >= NOW() - INTERVAL 30 DAY
> GROUP BY c.id, c.name
> ORDER BY total_amount DESC
> LIMIT 10;
> ```

The AI auto-detects tables and columns in your connected database — it won't generate references to tables or columns that don't exist.

### Explain SQL

Select a complex SQL query and ask the AI to explain it. It breaks down:

- What each subquery does
- The purpose of JOIN conditions
- What the WHERE clause filters
- How aggregation and grouping work

### Optimize SQL

The AI analyzes your query for potential performance issues and suggests improvements:

- Missing indexes
- Full table scans that could be avoided
- Subqueries that could be rewritten as JOINs
- Unnecessary sorting or deduplication

### Fix Errors

When a query fails, click **Fix with AI**. The AI reads the error message and your SQL, then generates a corrected version. Common fixes include:

- Syntax error corrections
- Misspelled table or column names
- Missing GROUP BY columns
- Type mismatches

## Beyond Built-in AI

<Callout type="info">
Want AI to not just generate SQL, but actually execute queries and browse results? Try the [MCP integration](/en/docs/mcp).
</Callout>

MCP integration lets AI coding agents like Claude Code and Cursor query your databases directly through DBX and open results in the DBX UI. You can have a natural language conversation with AI in your IDE, and it automatically calls DBX to run queries and return results.
