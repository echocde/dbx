<Callout type="info">
跨数据库结构对比，自动生成同步 SQL。无论是开发 vs 生产、还是不同环境之间的数据库结构对齐，一键搞定。
</Callout>

## 典型场景

- **开发 vs 生产** — 检查开发环境的 Schema 变更是否已同步到生产
- **多环境对齐** — 确保 staging、QA、production 的表结构一致
- **版本迁移** — 对比新旧版本数据库结构，生成迁移脚本

## 使用流程

<Steps>
  <Step>
    ### 选择源和目标
    分别选择**源数据库**和**目标数据库**连接。可以是不同的服务器，甚至不同的数据库引擎。
  </Step>
  <Step>
    ### 执行对比
    点击"对比"按钮，DBX 自动分析两个数据库中所有表和字段的差异。
  </Step>
  <Step>
    ### 审查差异
    对比结果以清晰的列表展示：
    - **仅在源库存在的表** — 目标库中缺少的表
    - **仅在目标库存在的表** — 源库中没有的表
    - **结构不同的表** — 字段类型、默认值、索引等存在差异的表
  </Step>
  <Step>
    ### 生成并执行同步 SQL
    DBX 根据差异自动生成 `CREATE TABLE`、`ALTER TABLE`、`DROP TABLE` 等 SQL 语句。确认无误后，可以直接在 DBX 中执行。
  </Step>
</Steps>

<Callout type="warn">
Schema 同步可能包含破坏性操作（如删除表或列）。执行前请仔细审查生成的 SQL，建议先备份目标数据库。
</Callout>
