<Callout type="info">
Cross-engine data migration with automatic type mapping and syntax adaptation. MySQL to PostgreSQL, SQLite to SQL Server — it just works.
</Callout>

## Use Cases

- **Cross-engine migration** — MySQL → PostgreSQL, SQLite → SQL Server
- **Environment sync** — Pull production data into your dev environment
- **Data consolidation** — Merge data from multiple sources into one database

## Workflow

<Steps>
  <Step>
    ### Select Source and Target
    Choose the **source** and **target** connections. They can be different database engines.
  </Step>
  <Step>
    ### Choose Tables
    Transfer all tables at once, or pick a subset.
  </Step>
  <Step>
    ### Start Transfer
    Click "Start Transfer". DBX handles type mapping and syntax differences automatically.
  </Step>
</Steps>

## Progress Tracking

During the transfer, a progress indicator shows:

- Which table is currently being transferred
- How many rows have been transferred so far
- The overall progress across all selected tables

## Error Handling

If an error occurs during transfer (type incompatibility, constraint violation, etc.), DBX:

- Logs the error with details about which table caused the problem
- Continues transferring other tables without stopping the entire job
- Shows a summary of all errors after the transfer completes
