Built on CodeMirror 6 with deep database metadata awareness. A SQL editing experience that rivals a full IDE.
</Callout>

## Syntax Highlighting

The editor applies language-aware syntax highlighting for SQL. Keywords, identifiers, strings, numbers, and comments are visually distinguished, making queries easier to read and write. Highlighting rules adapt automatically based on the connected database type.

## Autocomplete

As you type, the editor suggests completions drawn from your actual database metadata:

- **Keywords** — standard SQL keywords (`SELECT`, `FROM`, `WHERE`, etc.)
- **Table names** — all tables and views visible in the current connection
- **Column names** — columns scoped to the tables referenced in your query
- **JOIN suggestions** — when typing a `JOIN` clause, the editor suggests related tables and join conditions based on foreign key relationships

Completions appear automatically and can also be triggered manually with the platform shortcut.

## Executing Queries

<Callout type="info">
Shortcut: `Cmd+Enter` (macOS) / `Ctrl+Enter` (Windows/Linux). When text is selected, only the selection is executed.
</Callout>

| Action | Shortcut |
|---|---|
| Execute all SQL | `Cmd+Enter` / `Ctrl+Enter` |
| Execute selected SQL | Select text, then `Cmd+Enter` / `Ctrl+Enter` |

Results appear immediately below the editor in the Data Grid, showing returned rows, execution time, and affected row count.

## SQL Formatting

Use the format button in the toolbar to auto-format your SQL. The formatter standardizes indentation, keyword casing, and line breaks so that complex queries are easier to read and review.

## Query History

Every executed query is recorded in your local query history:

- Browse previously executed queries in the history panel
- Click any past query to load it back into the editor
- Search through past queries by keyword
- History is stored per connection — switching connections shows only the relevant history
