<Callout type="info">
跨数据库引擎数据迁移，自动处理类型映射和语法差异。MySQL 到 PostgreSQL，SQLite 到 SQL Server，开箱即用。
</Callout>

## 典型场景

- **跨引擎迁移** — MySQL → PostgreSQL、SQLite → SQL Server
- **环境数据同步** — 从生产库拉取数据到开发环境
- **数据整合** — 将多个数据源的数据汇集到一个数据库

## 使用流程

<Steps>
  <Step>
    ### 选择源和目标
    选择**源连接**和**目标连接**，可以是不同类型的数据库引擎。
  </Step>
  <Step>
    ### 选择要传输的表
    选择传输全部表，或手动勾选需要传输的特定表。
  </Step>
  <Step>
    ### 开始传输
    点击"开始传输"，DBX 自动处理类型映射和语法差异。
  </Step>
</Steps>

## 进度追踪

传输过程中，DBX 实时显示：

- 当前正在传输的表名
- 已传输的行数
- 整体传输进度

## 错误处理

如果传输过程中遇到错误（如类型不兼容、约束冲突等），DBX 会：

- 记录错误信息和对应的表名
- 继续传输其他表，不会因单个表的错误中断整个任务
- 传输完成后汇总展示所有错误，便于排查和处理
