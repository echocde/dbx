<Callout type="info">
Explore your database objects in a hierarchical tree with search, pin, column comments, and color-coded connections to tell dev from production at a glance.
</Callout>

## Tree View

The browser organizes objects in a hierarchical tree:

```
Connection
  └── Database
        └── Schema
              └── Tables / Views
                    └── Columns (type, constraints, defaults)
```

Expand any node to drill down. Click a table name to open it in the Data Grid. Column entries display their data type and attributes inline.

## Search and Pin

- **Search** — Type in the search field to filter across all connections, matching table names, view names, and column names
- **Pin** — Pin frequently used tables or views to the top of the sidebar for instant access, regardless of tree depth

## Context Menus

Right-click any node for context-specific actions:

- **Connection** — Refresh metadata, duplicate connection, disconnect
- **Database** — Create database, drop database
- **Schema** — Create schema, drop schema
- **Table** — Open data, open structure, copy table name, drop table
- **Column** — Copy column name

## Column Comments

If your database supports column comments (`COMMENT`), the Schema Browser displays them alongside each column, making it easy to understand column purposes without leaving the sidebar.

## Connection Colors

<Callout type="info">
Assign colors to connections — green for dev, red for production — to tell them apart at a glance and avoid accidental operations on the wrong environment.
</Callout>

Colors appear next to connection names in the sidebar and in tabs, especially useful when working with multiple connections.

## Redis Browser

The Redis browser provides a dedicated interface for browsing and managing Redis data:

- **Table View** — Keys are displayed in a Navicat-style table layout for easier scanning and management
- **Inline Editing** — Edit key values directly in the browser without switching to a separate editor
- **Pagination** — Browse large keyspaces with built-in pagination, avoiding performance issues from loading all keys at once
- **All Data Types** — Full support for String, Hash, List, Set, Sorted Set (ZSet), and Stream
