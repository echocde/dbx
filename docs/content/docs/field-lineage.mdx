<Callout type="info">
Most database tools don't have this. DBX automatically traces column relationships from foreign keys, views, and query history, with confidence levels — essential for understanding how data flows through your schema.
</Callout>

## Why Field Lineage?

- **Data governance** — Understand where each column's data comes from and ensure data quality
- **Impact analysis** — Before changing a table structure, see which downstream views or queries are affected
- **Onboarding** — Quickly understand relationships across tables in a complex database

## Relationship Sources

DBX discovers column relationships from multiple sources, each with a different confidence level:

### Foreign Key Relationships (Certain)

Relationships defined by foreign key constraints. For example, `orders.user_id → users.id`. These are authoritative and the most reliable source.

### View References (Likely)

DBX analyzes view definitions to trace which columns in the view map back to columns in the underlying tables.

### Query History References

DBX analyzes your local query history to identify columns that are frequently joined or referenced together in `JOIN` and `WHERE` clauses.

### Same-Name Column Detection

Columns that share the same name across different tables (e.g., `user_id` appearing in multiple tables) are flagged as potential relationships.

## Searching and Filtering

- **Search** — Filter relationships by table name or column name
- **Confidence filter** — Show only relationships at a specific confidence level (certain, likely, or all)

## Click-Through Navigation

Click any related table or column in the lineage view to navigate directly to it in the Data Grid or Schema Browser.
