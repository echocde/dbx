<Callout type="info">
大多数数据库工具没有这个功能。DBX 从外键、视图、查询历史等多个维度自动分析字段关联，标注置信度，帮你理解数据的完整流向。
</Callout>

## 为什么需要字段血缘？

- **数据治理** — 了解每个字段的数据来源，确保数据质量
- **影响分析** — 修改表结构前，了解哪些下游视图或查询会受影响
- **新人上手** — 快速理解复杂数据库中表与表之间的关系

## 关联来源

DBX 通过多种方式检测字段关联，每种方式标注不同的置信度：

### 外键关系（确定）

基于数据库中已定义的外键约束。例如 `orders.user_id → users.id`，这是最可靠的关联来源。

### 视图引用（高置信度）

分析视图定义中的 SQL 语句，提取视图字段与底层表字段的映射关系。

### 查询历史引用

从你执行过的 SQL 查询中，分析 `JOIN` 条件和 `WHERE` 子句，推断字段之间可能存在的关联。

### 同名字段检测

检测不同表中名称相同的字段（如多个表中的 `user_id`），提示它们之间可能存在逻辑关联。

## 搜索与筛选

- **搜索** — 输入表名或字段名，过滤显示相关的血缘关系
- **按置信度筛选** — 选择只查看特定置信度级别的关联（确定 / 高 / 全部）

## 跳转导航

在血缘视图中，点击任意关联的表或字段，可以直接跳转到该表的数据浏览或结构查看页面，方便你快速了解上下游数据。
